maybe_without_filename() -> Self

Return a new translatable string without the “filename” field.

Only acts upon translatable strings containing the exact contents ": {filename!r}". The specified part will be removed. This is correct usage in English for messages like "Cannot open file: {error}: {filename!r}.", but not necessarily in other languages.
